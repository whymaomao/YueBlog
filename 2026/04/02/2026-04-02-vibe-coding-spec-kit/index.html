Vibe Coding 第一课：Spec-Kit 思维训练

Posted on 2026-04-02 In ARTS

来源: Claude Code Vibe 教学 session | 日期: 2026-04-02

背景

Vibe coding 面试中，面试官评估的不是代码写得快不快，而是：
能否写出足够清晰的 spec，让 AI 第一次就交付正确的结果，并且自己能验证它。

核心知识点

Spec-Kit 的五个要素

在向 AI 提 prompt 之前，必须先写 spec-kit：

  1. 要构建什么 — 描述行为，不是实现
  2. 约束条件 — 技术栈、禁止使用的库、具体选型（不能模糊，AI 会自己填空）
  3. 边界情况 — 空结果、非法参数、超出范围的输入
  4. 期望的输出格式 — 具体的数据结构，每个字段都要写出来
  5. 验收条件 — 不是”用测试验”，而是”传入 X，返回结果必须满足 Y”

验收条件是最容易被跳过的一项

没有验收条件，你给 AI 的不是 spec，是祈祷。
验收条件要具体到输入输出的对应关系，而不是测试方式。

关键收获

  • Spec 不是需求说明，是你对系统行为的完整描述——写完之后你自己应该能验证它
  • 约束条件必须具体，否则 AI 会自己决定，选错了就要返工
  • 第一次写 spec-kit 一定有漏洞，迭代能力本身就是核心技能
  • Vibe coding 的核心不是”用 AI 写代码”，而是”写清晰到不需要手把手的 spec”
