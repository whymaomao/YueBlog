Terraform

Posted on 2024-08-05 Edited on 2026-03-27 In DevOps

The terraform plan command is a crucial part of Terraform’s workflow. It allows you to preview the changes that Terraform will make to your infrastructure before actually applying them. Here’s a step-by-step flow of what happens when you run terraform plan:

1. Load Configuration

  • Read Configuration Files: Terraform reads the .tf and .tf.json files in your working directory to understand the desired state of your infrastructure.
  • Parse and Validate Configuration: Terraform parses the configuration files and performs initial validation to check for syntax errors or misconfigurations.

2. Load State

  • Read State File: Terraform loads the current state file (terraform.tfstate) to understand the existing state of your infrastructure. This state file contains information about the resources that Terraform manages and their current configuration.
  • Synchronize with Backend: If you’re using a remote backend (like AWS S3, Azure Blob Storage, etc.), Terraform will fetch the latest state from the backend.

3. Refresh State

  • Query Infrastructure: Terraform queries the actual state of resources in the cloud provider (e.g., Azure, AWS) to get the current state of resources that are already deployed.
  • Update State File: The state file is updated with the current state information. This step ensures that Terraform’s view of the infrastructure is accurate before generating a plan.

4. Generate Execution Plan

  • Compare Desired and Current States: Terraform compares the desired state (as defined in your configuration files) with the current state (from the state file and actual infrastructure).
  • Create Plan: Terraform generates an execution plan that details the actions required to bring the infrastructure from its current state to the desired state. This includes creating, updating, or deleting resources.

5. Output Plan

  • Display Plan: Terraform outputs the plan to the terminal, showing a summary of the changes that will be made. This includes:
    • Additions: New resources that will be created.
    • Modifications: Existing resources that will be updated.
    • Deletions: Resources that will be destroyed.
  • Plan Details: For each resource, Terraform shows the current state and the planned changes.

6. User Review

  • Review the Plan: You review the output to ensure that the changes Terraform plans to make align with your expectations. This step is crucial to prevent unintended modifications.

7. Apply Plan

  • Manual Execution: After reviewing the plan, you can execute terraform apply to apply the changes. You may also apply the plan directly from a saved plan file if you use the terraform apply <plan-file> command.

Key Points

  • No Changes to Infrastructure: The terraform plan command does not make any changes to your infrastructure. It only generates a preview of what changes will occur if you decide to apply them.
  • Plan Files: You can save the plan to a file using the -out option (terraform plan -out=plan.tfplan) and apply it later using terraform apply plan.tfplan.

By following this flow, terraform plan helps ensure that you understand the implications of your changes before they are applied, reducing the risk of unintended modifications to your infrastructure.
